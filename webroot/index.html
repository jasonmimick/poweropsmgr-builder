MongoDB Ops Manager for IBM Power

PRE-GA DEVELOPMENT RELEASE


Introduction

Ops Manager is a package for managing MongoDB deployments. Ops Manager provides Ops Manager Monitoring and Ops Manager Backup, which helps users optimize clusters and mitigate operational risk.

This document will guide you through installing Pre-GA version of MongoDB Ops Manager on IBM POWER Linux. This Pre-GA version is for IBM internal purposes.

Please read this entire document before logging into any systems. Failure to properly perform all steps will result in a non-functional system.

Pre-Installation Notes

  1. You will need to use Local Mode for Ops Manager Servers even if your servers have internet access. This is due to the fact that this Pre-GA version does not yet support automatically downloading the correct ppc64le build of MongoDB Enterprise. In addition to manually downloading and unpacking the correct MongoDB Enterprise ppc64le builds you must also copy both the archive .tgz files and the folder created when unpacking, into similarily a similarly named file and directory. For example,
    cp ./mongodb-linux-ppc64le-enterprise-ubuntu1604-3.4.10.tgz ./mongodb-linux-x86_64-enterprise-ubuntu1604-3.4.10.tgz
    and
    cp -R mongodb-linux-ppc64le-enterprise-ubuntu1604-3.4.10 mongodb-linux-x86_64-enterprise-ubuntu1604-3.4.10
    Note, we have replaced ppc64le with x86_64.

Installation Instructions

  1. Download the lastest MongoDB Ops Manager IBM POWER archive from here or, just
    curl -OL http://54.89.245.135/builds/3.4.7.479-1/mongodb-mms-3.4.7.479-1.ppc64le.tar.gz

  2. Use the Install from Archive on Linux documentation to install Ops Manager using the archive downloaded in step 2. Be sure to configure Local Mode for Ops Manager when installing.
