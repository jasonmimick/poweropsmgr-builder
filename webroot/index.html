MongoDB Ops Manager for IBM Power

ALPHA DEVELOPMENT RELEASE


Before installing Ops Manager on and IBM POWER machine running Linux, prepare by validating the following:

Pre-Installation Procedure

  1. A full Java JDK version 1.8.0 is installed.
  2. Note the location of the JDK, while the post-install step will attempt to auto-detect the JDK location, you may need to enter this path to set the correct JAVA_HOME environment variable in order for Ops Manager to function properly.
  3. Decide if you wish to override the Ops Manager Local Version directory on your instance (/data/automation/mongodb-releases/). If so, note a path to which the user running Ops Manager has access. In the post-install step you will be able to set this configuration parameter (setting the automation.versions.directory) property.

Installation Instructions

  1. Download the lastest MongoDB Ops Manager IBM POWER archive from here or, just
    curl -OL http://54.89.245.135/builds/3.4.7.479-1/mongodb-mms-3.4.7.479-1.ppc64le.tar.gz
  2. Use the Install from Archive on Linux documentation to install Ops Manager, BUT....
  3. After step 2 (unpacking the Ops Manager archive) download and execute the post install script:
    curl -OL https://raw.githubusercontent.com/jasonmimick/poweropsmgr-builder/master/post-install.sh
    chmod +x post-install.sh
