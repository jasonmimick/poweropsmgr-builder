MongoDB Ops Manager for IBM Power

ALPHA DEVELOPMENT RELEASE


Introduction

Installing MongoDB Ops Manager on the IBM Power platform is not complicated. In general you'll be following the standard MongoDB documentation for installing Ops Manager. There are, however, a couple of steps you need to take. This document will guide you through the installation process.

Please read this entire document before logging into any systems. There is a cruicial step documented here which occurs after the 2nd step in the offical documentation. Failure to properly perform this step will result in a non-functional system.

Pre-Installation Procedure

  1. Validate a full Java JDK version 1.8.0 is installed. For example,
    $javac -version
  2. Note the location of the JDK, while the post-install step will attempt to auto-detect the JDK location, you may need to enter this path to set the correct JAVA_HOME environment variable in order for Ops Manager to function properly. One way to find your JAVA_HOME is,
    $readlink -f `which javac`
  3. Do you need to override the Ops Manager Local Version directory?
    The default is /data/automation/mongodb-releases/).
    If NO continue to step 4, else continue.

    Note the path you wish to use, then ensure the user you run the mongodb-mms service as has read and write permission on that path. In the post-install step you will be able to set this configuration parameter (automation.versions.directory) .
  4. You will need to manually download the IBM POWER (ppc64le) builds for each MongoDB version you wish to backup with Ops Manager.
    Be sure to place the .tgz file in the automation.versions.directory directory and ALSO unpack the archive!

Installation Instructions

  1. Download the lastest MongoDB Ops Manager IBM POWER archive from here or, just
    curl -OL http://54.89.245.135/builds/3.4.7.479-1/mongodb-mms-3.4.7.479-1.ppc64le.tar.gz
  2. Use the Install from Archive on Linux documentation to install Ops Manager, BUT....
  3. After step 2 (unpacking the Ops Manager archive) download and execute the post install script:
    curl -OL https://raw.githubusercontent.com/jasonmimick/poweropsmgr-builder/master/post-install.sh
    chmod +x post-install.sh
    ./post-install.sh
